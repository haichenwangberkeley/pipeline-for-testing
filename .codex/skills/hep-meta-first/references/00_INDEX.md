# Structured Skill Index

This index organizes the refactored contracts in `newskills/` for function-like skill invocation planning.

## Planning Semantics

- Use `outputs` in each skill contract to answer: "Which skill produces this output?"
- Use `inputs` + `preconditions` + `dependencies.requires` to answer: "Which skill can run now?"
- Use `dependencies.requires` and `handoff_to` to answer: "Which skill must run first?"
