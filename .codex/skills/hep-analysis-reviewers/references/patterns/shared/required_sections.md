# Required Sections

Every skill should include these sections:

- Purpose
- When to use
- Inputs
- Outputs
- Preconditions
- Postconditions
- Procedure, decision structure, review scope, or stage table depending on pattern
- Evidence or logging requirements
- Failure modes
- Escalation
- Related skills

Pattern-specific additions:

- Tool Wrappers: command or module entrypoints plus verification expectations
- Generators: output contract plus constraints
- Reviewers: pass, conditional-pass, block, and fail criteria
- Inversions: explicit branches plus decision record fields
- Pipelines: stage ordering plus required reviewer gates
