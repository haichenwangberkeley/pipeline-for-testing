# Common Reviewer Rubric

Reviewers in this skill pack use four outcomes:

- `pass`: evidence is complete and the stage may advance
- `conditional_pass`: minor issues remain but are logged and non-blocking
- `block`: required information, policy compliance, or scientific validity is insufficient
- `fail`: the output is incorrect, inconsistent, or unsafe for downstream use

## Review dimensions

- Scientific validity
- Artifact completeness
- Provenance and reproducibility
- Policy compliance
- Reporting clarity
- Handoff readiness

## Reviewer behavior

- Cite the artifact or log that supports each finding.
- Distinguish implementation bugs from unresolved modeling limitations.
- Recommend the smallest next skill that can remediate the issue.
- Escalate to a human when the missing evidence cannot be reconstructed locally without inventing physics content.
