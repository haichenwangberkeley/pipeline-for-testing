## Execution Policy

- Load and use the skills relevant to the current task.
- Treat all applicable skills as binding contracts.
- Execute the work in explicit, sequential steps.
- At every step:
  1. inspect the current state,
  2. evaluate compliance with the skills,
  3. fix any non-compliance,
  4. then execute.
- Never assume an existing pipeline, script, configuration, or artifact is correct or skill-compliant without checking.
- When there is a conflict between existing implementation and skills, the skills take precedence.
- Continue until the full task is completed.
